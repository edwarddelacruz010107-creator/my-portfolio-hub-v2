THE FIX IN ONE SENTENCE:
================================================================================

Disable WTF_CSRF_SSL_STRICT for login routes only (they're CSRF-safe via SOP),
while keeping it enabled for all other routes in production.

Root cause: Render.com proxy causes request.host to be internal hostname,
causing CSRF validation to fail before credential checking.
