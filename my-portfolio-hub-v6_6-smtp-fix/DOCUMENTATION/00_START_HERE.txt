🔐 SECURITY FEATURES
═════════════════════════════════════════════════════════════════════════════

✅ Encryption
   → API keys encrypted with Fernet (symmetric, key from Flask.secret_key)
   → Decrypted only in Python, never sent to browser

✅ Password Fields
   → Never pre-filled, never echoed in responses
   → Password field behavior for API key input

✅ Access Control
   → Superadmin-only endpoint
   → CSRF token required on all forms
   → Session validation on every request

✅ Validation
   → Prevents incomplete configurations
   → All-or-nothing approach (all 3 fields or none)

✅ Audit Trail
   → updated_at timestamp on configuration changes
   → Legacy SMTP fields retained for historical reference

✅ Rollback Safety
   → Migration downgrade drops only new columns
   → Original SMTP data untouched
   → Zero data loss on rollback

🔄 ROLLBACK PROCEDURE
═════════════════════════════════════════════════════════════════════════════

If something goes wrong, rollback in one command:

  flask db downgrade -1

This will:
  ✓ Drop the three new columns cleanly
  ✓ Revert to previous schema
  ✓ Leave SMTP data intact (never touched)
  ✓ Restore old templates (via git)

NO data loss. Safe rollback at any time.
