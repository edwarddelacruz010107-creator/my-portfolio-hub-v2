// 01 — about

A little about how I work

I'm a backend-leaning full-stack developer who spends most of my time in Flask, SQLAlchemy, and whatever database is causing problems that week.

I started out fixing WordPress sites for local businesses, which taught me more about defensive coding than any course did — nothing sharpens error handling like a client calling at 11pm. These days I mostly build multi-tenant platforms, billing systems, and internal tools that need to survive contact with real users.

I'm not precious about tech choices. I pick boring, well-understood tools for anything load-bearing, and save the experimentation for the parts of a project that can afford to fail.
